# Пневмоклешня с управлением через веб

Учебный проект по предмету ОВТИС. Роботизированная клешня из двух пальцев, которая хватает и отпускает предметы. Пальцы сводятся не мотором, а пневматической мышцей — той же технологией, что используют в мягкой робототехнике. Через браузер можно подать команду, и клешня сожмётся или раскроется.

В этом репозитории лежит программная часть: сервер, веб-панель и демонстрация. Сам механизм собран отдельно и напечатан на 3D-принтере.

## Быстрый старт

Нужен установленный Python 3.

```
pip install -r requirements.txt
python server.py
```

Сервер запустится, после чего открываем браузер по адресу `http://127.0.0.1:5000`. Откроется панель — кнопки «Сжать клешню» и «Разжать», а также ползунок силы сжатия. Если нужно зайти с телефона, который в той же сети, — вводим IP компьютера вместо `127.0.0.1`.

Если железа под рукой нет, а показать интерфейс надо — открываем `demo.html` обычным двойным кликом. Это упрощённая версия, она крутится прямо в браузере и сервер ей не нужен.

## Как устроено управление

Важная идея проекта в том, что **думает не браузер, а сервер**. Браузер — это просто пульт: ты нажал кнопку, он отправил команду и нарисовал ответ. А вот где реально хранится, открыта клешня или закрыта, какое сейчас давление — это всё на сервере.

Цепочка получается такая:

```
кнопка в браузере  ->  HTTP-запрос  ->  сервер на Python  ->  (в железе) клапан и мышца
                                          сервер помнит состояние
                                          и присылает его назад
```

Почему так, а не просто переключать картинку скриптом: если состояние живёт только в браузере, то стоит обновить страницу или открыть её на другом устройстве — и всё «забывается», потому что сервер ничего не знал. Когда состояние на сервере, оно одно на всех и не теряется. Браузер раз в секунду переспрашивает сервер («как сейчас дела?») и обновляет показания — поэтому панель всегда отражает реальное положение.

В готовом изделии вместо Python-сервера стоит микроконтроллер ESP32. Он отвечает на те же самые запросы и уже физически открывает клапан, пускает воздух от компрессора в мышцу. Логика одна и та же, поэтому веб-панель для обоих случаев не меняется.

## Как работает сама клешня

Привод здесь необычный — **пневматическая мышца Маккиббена**. Это эластичная камера в плетёном чехле. Накачиваешь в неё воздух — она толстеет и одновременно становится короче. Сбросил воздух — мышца обмякла, пальцы разошлись.

Получается поведение, очень похожее на живую мышцу: давит мягко, без жёстких передач и моторов. Чем больше накачано — тем сильнее сжатие.

## Список команд сервера

Панель общается с сервером простыми запросами:

- `/close` — накачать мышцу, клешня сжимается
- `/open` — стравить воздух, клешня раскрывается
- `/status` — узнать текущее состояние; ответ приходит в JSON, например:

```json
{ "closed": true, "state": "сжата" }
```
