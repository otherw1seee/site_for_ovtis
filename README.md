# Роботизированная клешня на пневматической мышце Маккиббена

Курсовой проект: роботизированная клешня из двух пальцев, которая сжимается и разжимается с помощью пневматической мышцы Маккиббена. Управление — через веб-панель с серверной обработкой на Python.

Проект состоит из двух частей: **механической** (сама клешня и пневмопривод) и **информационной** (управление с сервера).

---

## 1. Механическая часть

### Принцип работы

Клешня приводится в движение **пневматической мышцей Маккиббена**. Это резиновая трубка в плетёной оплётке. Когда компрессор нагнетает внутрь воздух, мышца раздувается в диаметре и при этом **укорачивается** по длине — то есть тянет, как биологическая мышца. Через сухожилия (тросы) это укорочение сводит два пальца клешни.

Чем выше давление в мышце — тем сильнее она сокращается и тем сильнее сжимается клешня. При сбросе давления мышца расслабляется, и пальцы расходятся.

### Особенности

- Привод не электромеханический (как серво или мотор), а **пневматический** — это ближе к тому, как работают настоящие мышцы.
- Клешня и детали каркаса напечатаны на 3D-принтере.
- Одна мышца управляет сведением двух пальцев клешни.

## 2. Информационная часть (управление)

### Архитектура

```
браузер (веб-панель)  ->  сервер (Python / ESP32)  ->  клапан / компрессор  ->  мышца
                       <-  состояние (давление)
```

Управление построено по принципу клиент-сервер. Состояние системы (текущее давление в мышце, сжата ли клешня) хранится и обрабатывается **на сервере**, а не в браузере. При нажатии кнопки или движении ползунка браузер отправляет HTTP-запрос на сервер, сервер обрабатывает его, меняет состояние и возвращает обновлённый статус. Браузер только показывает то, что вернул сервер.

Это важное отличие от простой страницы: если бы кнопка переключалась только в браузере на JavaScript, сервер не знал бы о состоянии — это была бы лишь видимость управления. Здесь состояние общее и серверное.

В реальном изделии роль сервера выполняет контроллер **ESP32**: он принимает те же запросы и управляет электроклапаном или компрессором (например, ШИМ-сигналом), регулируя давление в мышце. Для демонстрации без железа сервер написан на Python (Flask) — он повторяет ту же логику на компьютере.

### Возможности панели

- **Сжать клешню / Разжать** — накачать мышцу до полного давления или стравить воздух.
- **Регулировка силы** — ползунок давления от 0 до 100 %, задаёт силу сжатия клешни.
- **Схема в реальном времени** — вид клешни, два пальца которой сводятся по мере роста давления, и мышца, которая раздувается.
- **Телеметрия** — текущее давление и состояние (раскрыта / сжимается / сжата).

### Запуск сервера

```
pip install -r requirements.txt
python server.py
```

После запуска открыть в браузере: http://127.0.0.1:5000
С другого устройства в той же сети — по IP компьютера: `http://<IP>:5000`

### Запросы к серверу (API)

| Запрос | Действие |
|---|---|
| `/close` | сжать клешню — накачать мышцу (давление 100 %) |
| `/open` | разжать клешню — стравить воздух (давление 0 %) |
| `/pressure?value=N` | задать давление в мышце (0…100 %) — регулировка силы сжатия |
| `/status` | получить текущее состояние в формате JSON |

Пример ответа `/status`:

```json
{ "pressure": 100, "closed": true, "state": "сжата" }
```

## Файлы

- `server.py` — сервер на Flask: хранит состояние, обрабатывает команды, раздаёт панель
- `index.html` — веб-панель управления (рабочая версия, общается с сервером)
- `demo.html` — упрощённая демонстрация интерфейса (работает в браузере без сервера)
- `requirements.txt` — зависимости сервера
- `README.md` — это описание

## Возможные доработки

- Подключить датчик давления, чтобы отображать реальное давление в мышце, а не заданное.
- Заменить Python-сервер на контроллер ESP32 по тем же адресам — код панели менять не нужно.
- Добавить кнопку аварийного сброса давления.
